## このマニュアルについて

本マニュアルは、Cafe Lore LINE公式アカウントの日常運用・メンテナンスに必要な手順をまとめたものです。主にカフェスタッフ、または運用を担当される方が対象です。

技術的な作業（コードの編集やデプロイ）が含まれる項目には「要：PC作業」と表記しています。PC作業が難しい場合は、開発担当者に依頼してください。

## LINE公式アカウントの全体像

Cafe LoreのLINE公式アカウントは、以下の仕組みで動いています。

```
お客様のLINE → LINE Messaging API → Vercel（サーバー） → 応答を返す
                                         ↓
                                    Claude AI（必要な場合のみ）
```

お客様がメッセージを送ると、Vercel上のプログラムが受け取り、内容に応じて以下のいずれかで応答します。

- よくある質問（営業時間、Wi-Fiなど） → あらかじめ用意した定型文で即座に回答
- 「メニュー」「予約」などのキーワード → カード形式で見やすく表示
- それ以外の質問 → AIコンシェルジュ（Claude）が店舗情報をもとに回答

## LINE公式アカウント管理画面での設定

### アカウントの基本設定

LINE Official Account Manager（https://manager.line.biz/ ）で以下を管理できます。

- プロフィール画像の変更
- ステータスメッセージの変更（「代官山のスペシャルティコーヒーカフェ」など）
- 背景画像の変更
- あいさつメッセージの設定（Bot側で送信しているためオフ推奨）

### 応答設定の確認

以下の設定が正しいことを必ず確認してください。これが間違っているとBotが正常に動きません。

1. LINE Official Account Manager にログイン
2. 「設定」→「応答設定」を開く
3. 以下の状態になっていること:
   - 応答モード: **Bot**
   - あいさつメッセージ: **オフ**
   - 応答メッセージ: **オフ**
   - Webhook: **オン**

### LINE Developers Console側の設定

https://developers.line.biz/ の Messaging API設定タブで以下を確認:

- **Webhook URL**: `https://line-ai-bot-xxx.vercel.app/api/webhook`（実際のVercel URLに読み替え）
- **Webhook の利用**: オン
- **チャネルアクセストークン**: 発行済みであること（有効期限なし）

### チャネルアクセストークンの再発行が必要な場合

トークンが漏洩した疑いがある場合や、セキュリティ上の理由で再発行が必要な場合:

1. LINE Developers Console → チャネル → Messaging API設定
2. 「チャネルアクセストークン（長期）」の「再発行」をクリック
3. 新しいトークンをコピー
4. Vercelの環境変数 `LINE_CHANNEL_ACCESS_TOKEN` を新しい値に更新
5. Vercelで再デプロイ（手順は次章参照）

## Vercelでのサーバー管理

### Vercelとは

Cafe LoreのLINE Botプログラムをホスティングしているサービスです。Vercel Dashboard（https://vercel.com/dashboard ）からログインして管理します。

### 環境変数の管理

Botが動作するために3つの環境変数（秘密の設定値）が必要です。

| 変数名 | 説明 | 取得元 |
|--------|------|--------|
| `LINE_CHANNEL_SECRET` | LINEチャネルの認証に使うシークレットキー | LINE Developers Console → チャネル基本設定 |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE APIを呼び出すためのトークン | LINE Developers Console → Messaging API設定 |
| `ANTHROPIC_API_KEY` | AIコンシェルジュ（Claude）のAPIキー | Anthropic Console（https://console.anthropic.com/ ） |

#### Vercel Dashboardから変更する場合

1. https://vercel.com/dashboard にログイン
2. Cafe Loreのプロジェクトを選択
3. 上部メニューの「Settings」をクリック
4. 左メニューの「Environment Variables」をクリック
5. 変更したい変数の鉛筆アイコンをクリック
6. 新しい値を入力して保存
7. **重要**: 変更を反映するために再デプロイが必要（次項参照）

#### コマンドラインから変更する場合（要：PC作業）

```bash
cd ~/line-ai-bot
vercel env rm LINE_CHANNEL_ACCESS_TOKEN
vercel env add LINE_CHANNEL_ACCESS_TOKEN
# プロンプトに従って値を入力
```

### デプロイ（変更の反映）

コードや環境変数を変更した場合、Vercelに反映させる必要があります。

#### Vercel Dashboardから再デプロイ

1. Vercel Dashboard → Cafe Loreプロジェクト
2. 「Deployments」タブを開く
3. 最新のデプロイの右側「...」メニューをクリック
4. 「Redeploy」を選択
5. 数十秒で完了

#### コマンドラインからデプロイ（要：PC作業）

```bash
cd ~/line-ai-bot
vercel --prod
```

### Vercelのログ確認

Botの動作に問題がある場合、ログを確認できます。

1. Vercel Dashboard → プロジェクト → 「Logs」タブ
2. エラーが赤色で表示される
3. 「Function」フィルタで `api/webhook` を選択すると、Bot関連のログだけ表示

## リッチメニューの更新方法

### 現在のリッチメニュー構成

画面下部に表示される6ボタンのメニューです。2行3列のレイアウトになっています。

```
┌──────────────┬──────────────┬──────────────┐
│   AI 相談    │   メニュー   │    予約      │
│              │              │              │
├──────────────┼──────────────┼──────────────┤
│   クーポン   │   アクセス   │ お問い合わせ │
│              │              │              │
└──────────────┴──────────────┴──────────────┘
```

### リッチメニュー画像の差し替え（要：PC作業）

季節やキャンペーンに合わせてリッチメニューの見た目を変更できます。

1. 2500 x 1686ピクセルのPNG画像を用意（6分割を意識したデザイン）
2. プロジェクトフォルダに画像を配置
3. 以下のコマンドを実行:

```bash
cd ~/line-ai-bot
npm run setup-richmenu -- --image ./新しい画像名.png
```

4. LINE アプリでトーク画面を開き直して確認

画像生成スクリプトで自動作成することもできます:

```bash
node scripts/generate-richmenu-image.js
# → richmenu.png が生成される
npm run setup-richmenu -- --image ./richmenu.png
```

### ボタンの配置やアクションを変更する場合（要：PC作業）

`scripts/setup-richmenu.js` を編集します。`RICH_MENU.areas` 配列の各要素が1つのボタンに対応しています。

変更例: ボタンのラベルやアクションを変える場合

```js
{
  bounds: { x: 0, y: 0, width: 833, height: 843 },
  action: {
    type: "postback",
    label: "ボタンに表示するテキスト",
    data: "action=処理名",
    displayText: "お客様のチャットに表示されるテキスト",
  },
}
```

変更後は画像も更新して、再度セットアップコマンドを実行してください。

### リッチメニューが反映されない場合

- LINEアプリを閉じて再起動
- トーク画面を閉じて開き直す
- それでも変わらない場合は、友だちを削除して再追加（テスト用アカウントで確認推奨）

## メニュー表示内容の変更方法

### 編集するファイル（要：PC作業）

`lib/flex-messages.js` がメニューカードの表示内容を定義しています。

### ドリンクメニューを変更する

`drinkCard()` 関数の `items` 配列を編集します。

```js
items: [
  ["商品名", "¥価格"],
  ["ドリップコーヒー", "¥480"],
  ["カフェラテ", "¥550"],
  // 追加・削除・価格変更はここで行う
],
```

オプション料金（アイス +50円など）は `note` プロパティで変更:

```js
note: "Ice +¥50 / ミルク変更 +¥80 / デカフェ +¥50",
```

### フードメニュー、デザートメニューを変更する

同じファイル内の `foodCard()` と `dessertCard()` をそれぞれ編集します。構造はドリンクと同じです。

### 予約案内の内容を変更する

`reservationMessage()` 関数内の `infoRow()` で各項目を変更します。

```js
infoRow("営業時間", "平日 8:00〜21:00\n土日祝 9:00〜20:00"),
infoRow("定休日", "毎週水曜日"),
infoRow("席数", "店内22席 + テラス8席"),
infoRow("電話", "03-6455-XXXX"),
```

### カード全体のデザイン（色）を変更する

ファイル冒頭のカラーパレット `C` を編集します。

```js
const C = {
  bg: "#F2F0EB",        // 背景色（ベージュ）
  accent: "#1E3932",    // メインカラー（ダークグリーン）
  accent2: "#00704A",   // サブカラー（グリーン）
  accent3: "#6B4226",   // サブカラー（ブラウン）
  // ...
};
```

### 変更後の反映手順

1. ファイルを保存
2. デプロイを実行: `vercel --prod`
3. LINEで「メニュー」と送信して表示を確認

## AI応答の調整方法

### 編集するファイル（要：PC作業）

`lib/claude.js` の `SYSTEM_PROMPT` がAIコンシェルジュの「知識」と「振る舞い」を定義しています。

### システムプロンプトの構成

AIは以下の情報をもとに回答しています。すべて `SYSTEM_PROMPT` 内に記述されています。

| セクション | 内容 | 更新頻度の目安 |
|-----------|------|---------------|
| 店舗情報 | 住所、営業時間、設備、電話番号 | 変更があったとき |
| ドリンクメニュー | 全ドリンクの名前・価格・説明 | メニュー変更時 |
| フードメニュー | ランチ・終日メニュー | メニュー変更時 |
| デザートメニュー | デザートの名前・価格・説明 | メニュー変更時 |
| 季節限定メニュー | 期間限定メニュー | 季節ごと（年4回） |
| アレルギー情報 | アレルゲン別の対象メニュー | メニュー変更時 |
| 予約について | 予約方法・キャンセルポリシー | 変更があったとき |
| クーポン情報 | 配信中のクーポン | キャンペーン変更時 |
| スタッフ | スタッフ紹介 | 変更があったとき |
| イベント | 定期イベント情報 | イベント変更時 |
| 応答ルール | 回答の長さ・トーン・制約 | 基本変更不要 |

### よくある更新作業

#### 季節限定メニューを更新する

```
## ドリンクメニュー
...
【季節限定（3〜4月）】
- 桜ラテ ¥650 — 桜の花びらシロップ＋ミルク
```

この部分を新しい季節のメニューに書き換えます。

#### 日替わりメニューの説明を更新する

```
- 本日のスープ ¥500 — 日替わり。今日はかぼちゃのポタージュ
```

「今日は〜」の部分を当日のメニューに書き換えます。毎日の更新が理想ですが、週1回程度でも問題ありません。AIは「日替わりです」と答えることができます。

#### クーポンを追加・変更する

```
## クーポン情報
- LINE友だち追加特典: 初回ドリンク10%OFF（7日間有効）
```

この一覧に追加・削除します。`api/webhook.js` のFAQ応答にあるクーポン文面も合わせて更新してください。

#### AIの応答トーンを変える

`応答ルール` セクションを書き換えます。例:

- 「絵文字は控えめに」→「絵文字を積極的に使って親しみやすく」
- 「5文以内」→「3文以内でよりコンパクトに」

### AIモデルの設定

```js
model: "claude-haiku-4-5-20251001",  // 使用するAIモデル
max_tokens: 512,                     // 回答の最大長
```

- Haiku: 高速・低コスト。通常運用はこれで十分
- max_tokens: 512で通常のカフェ応答には十分。長い説明が必要な場合は768〜1024に増やす

### 変更後の反映手順

1. ファイルを保存
2. デプロイを実行: `vercel --prod`
3. LINEで質問を送信して回答内容を確認

## FAQ自動応答の管理

### 仕組み

お客様のメッセージに特定のキーワードが含まれていた場合、AIを呼ばずにあらかじめ用意した定型文で即座に回答します。これにより応答速度が上がり、AIのAPI利用コストも抑えられます。

### 編集するファイル（要：PC作業）

`api/webhook.js` の `FAQ_ENTRIES` 配列がキーワードと応答の定義です。

### FAQ項目の追加方法

```js
{
  keywords: ["トリガーワード1", "トリガーワード2"],
  answer: "お客様に返す応答テキスト",
},
```

例: 新しいFAQを追加する場合

```js
{
  keywords: ["禁煙", "タバコ", "喫煙"],
  answer: "当店は全席禁煙です。お近くの喫煙所をご案内しますので、スタッフにお声がけください。",
},
```

### キーワードの設計のコツ

- お客様が使いそうな言葉を幅広く登録する（例: 「Wi-Fi」「WiFi」「ワイファイ」）
- 短すぎるキーワードは誤マッチに注意（例: 「席」だけだと「お席の予約」にもマッチしてしまう）
- 迷ったら短いキーワードは避け、AIに任せる

### 現在登録されているFAQカテゴリ

1. 営業時間
2. アクセス・場所
3. Wi-Fi・電源
4. ペット同伴
5. 駐車場
6. クーポン・割引
7. アレルギー対応
8. テイクアウト
9. お子様連れ
10. バースデー・ケーキ持ち込み
11. 混雑状況
12. コーヒー豆の販売
13. 個室
14. 貸切・パーティー
15. 決済方法

### FAQ応答の末尾にはクイックリプライボタンが付く

すべてのFAQ応答の後に、以下の3つのボタンが表示されます:

- 「メニューを見る」
- 「予約する」
- 「AIに相談」

これによりお客様の次のアクションを促します。

## トラブルシューティング

### Botが全く応答しない

確認手順:

1. LINE Official Account Manager → 応答設定 → Webhookがオンか確認
2. LINE Developers Console → Messaging API設定 → Webhook URLが正しいか確認
3. Vercel Dashboard → Logs → エラーが出ていないか確認
4. 環境変数が正しく設定されているか確認（特にトークンの前後にスペースが入っていないか）

### 「ただいま応答できませんでした」と返ってくる

これはAIへの問い合わせでエラーが発生した場合のメッセージです。

- Anthropic APIキーが有効か確認（https://console.anthropic.com/ でログイン）
- APIの利用制限（レートリミット）に達していないか確認
- Vercelのログで具体的なエラーメッセージを確認

### FAQ応答が返らず、AIが回答してしまう

- `api/webhook.js` の `FAQ_ENTRIES` に該当キーワードが登録されているか確認
- キーワードの大文字・小文字を確認（自動で小文字比較しているが、日本語は完全一致）
- メッセージの前後の空白は自動的に除去されるが、キーワードが文中に含まれているか確認

### リッチメニューが表示されない

1. `npm run setup-richmenu -- --image ./richmenu.png` を再実行
2. LINEアプリを完全に閉じて再起動
3. トーク画面を閉じて開き直す
4. 上記で解決しない場合: 友だちを削除 → 再追加

### メニューカード（Flex Message）が表示されない

- LINEアプリが最新版になっているか確認
- 古い端末やPC版LINEでは一部のFlex Messageが表示されないことがある

### デプロイしたのに変更が反映されない

- Vercel Dashboardで最新デプロイのステータスが「Ready」になっているか確認
- 環境変数を変更した場合は必ず再デプロイが必要
- ブラウザで `https://xxx.vercel.app/api/webhook` にアクセスして `{"status":"ok"}` が返れば正常

### AIの回答が不適切・間違っている

- `lib/claude.js` の `SYSTEM_PROMPT` に正しい情報が記載されているか確認
- 情報が古い場合は更新してデプロイ
- 応答ルールを調整して回答の精度を改善

## 緊急時の対応

### Botを一時停止したい場合

LINE Developers Console → Messaging API設定 → 「Webhookの利用」をオフにすると、Botの応答が即座に停止します。復旧時はオンに戻すだけです。

### APIキーが漏洩した疑いがある場合

1. **Anthropic APIキー**: https://console.anthropic.com/ でキーを無効化し、新しいキーを発行
2. **LINEチャネルアクセストークン**: LINE Developers Console で再発行
3. Vercelの環境変数を更新
4. 再デプロイ

### 想定外のメッセージが大量に来た場合（スパム等）

- LINEの応答は1メッセージにつき1回のみなので、サーバー負荷は限定的
- Vercelの無料枠を超える場合は、Webhookを一時オフにして対応
- 継続的な場合はLINE Official Account Manager からブロック
